# 上课内容与Lexicon匹配方案设计

## 背景与需求

### 场景描述
- 输入：50-100段上课内容（每段约200字）
- Lexicon库：500个关键词/短语（如"油气工业"、"油气储量"等）
- 目标：为每段上课内容匹配最相关的Lexicon元素

### 核心约束
1. **唯一性**：每段上课内容至多匹配一个Lexicon元素（可以没有匹配）
2. **阈值过滤**：匹配相似度需超过设定阈值（初始0.5）
3. **LLM验证**：匹配成功后需经LLM二次验证，输出匹配程度、理由等

## 技术方案

### 1. 整体架构

### 2. 关键设计决策

#### 2.1 Lexicon入库 vs 段落入库

**推荐：Lexicon入库，段落作为查询**

理由：
- 需求是"每段上课内容匹配一个Lexicon"，即**给定段落，找Lexicon**
- 把Lexicon预计算成向量存入库（如FAISS）
- 每段上课内容转成向量，做**最近邻搜索（ANN）**，直接取Top-1
- 时间复杂度从O(N×M)降到O(N×logM)

#### 2.2 段落切分策略

**建议：按句子切分，不要整段一起embedding**

原因：
- 如果整段中只有2句话相关，其余8句完全不相关，整体语义向量会被"稀释"
- 按句子切分后，每个句子单独匹配，可以更精准地找到相关内容

**切分方法**：
```python
import re

def split_into_sentences(text):
    # 中文句子切分（按句号、问号、感叹号）
    sentences = re.split(r'[。！？\n]+', text)
    # 过滤空字符串和过短的句子
    sentences = [s.strip() for s in sentences if len(s.strip()) > 10]
    return sentences
```

#### 2.3 Embedding模型选择

**推荐：BAAI/bge-large-zh-v1.5**

理由：
- 中文效果优秀，在多个中文NLP任务上表现领先
- 开源免费，可本地部署
- 向量维度1024，平衡了表达能力和计算开销

备选：
- **OpenAI text-embedding-3-large**：多语言能力强，API调用方便，但有调用成本
- **M3E-base/large**：轻量级，推理速度快，适合对延迟敏感的场景

#### 2.4 阈值设定策略

**初始阈值：0.5**（余弦相似度范围[-1,1]，0.5约等于60度夹角）

**后续优化建议**：
- 准备一批标注数据（人工判断哪些段落应该匹配哪些lexicon）
- 计算不同阈值下的Precision、Recall、F1，选择最优阈值
- 可以设置**自适应阈值**：比如取Top-1相似度的80%作为阈值

#### 2.5 LLM验证策略

**验证目标**：让LLM判断匹配程度、给出理由

**Prompt设计建议**：
```
【任务描述】
判断以下上课内容段落是否与指定的Lexicon概念相关，并给出匹配程度评估。

【输入信息】
段落内容：{sentence}
匹配Lexicon：{lexicon}
相似度分数：{similarity}

【输出要求】
请按以下JSON格式输出：
{{
  "is_match": "是/否",
  "match_degree": "强相关/弱相关/不相关",
  "reason": "判断理由...",
  "confidence": 0.85
}}

注意：
1. is_match为"是"表示段落确实在讲解该Lexicon概念
2. match_degree表示匹配程度，强相关表示核心内容，弱相关表示边缘提及
3. reason需要具体说明段落中的哪些内容与Lexicon相关
4. confidence是置信度分数，范围[0,1]
```

## 3. 性能预估

**场景**：100个段落，500个lexicon

**计算量**：
- 编码100个段落：~100次embedding调用
- 编码500个lexicon：~500次embedding调用（可缓存复用）
- 向量搜索：100次FAISS查询，每次O(log 500)
- LLM验证：假设30个通过阈值，30次LLM调用

**预估耗时**：
- Embedding编码：~5-10秒（取决于模型和硬件）
- FAISS搜索：~0.1秒
- LLM验证：~10-30秒（取决于LLM响应速度）

**总计**：约20-40秒，符合"秒级"要求

## 4. 总结与建议

你的方案整体思路是**正确的**，主要设计决策都合理：

✅ **Lexicon入库、段落查询**：符合需求，性能更优  
✅ **按句子切分**：避免语义稀释，精准匹配  
✅ **阈值0.5**：合理起点，后续可优化  
✅ **LLM二次验证**：提升匹配质量，输出可解释性

3. 根据测试结果调整阈值和Prompt
4. 再逐步优化性能（加缓存、FAISS等）
